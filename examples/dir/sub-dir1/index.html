The problem of trailing slash - a subtle problem

If the URL does not include the filename, but only the directory, the server will return the default index.html file of that sub-dir.

It is not a 301 redirect, this action is taken quietly by the server. The browser knows nothing about that.

But the URL for sub-dir can be ambiguous. It can have or not have the trailing slash.

1st example: /sub-dir1/

2nd example: /sub-dir1

Both of these will open the same page /sub-dir/index.html and it looks fine.

But

If this page contains relative links, these 2 examples will behave differently !

For example this relative link:

another.html

The 1st example (trailing slash) will open the page: /sub-dir1/another.html. This is expected.

The 2nd example (not trailing slash) would open the page: /another.html. What? How?

The story goes that the browser joined '/sub-dir1' and 'another.html'.

The browser doesn't know that sub-dir1 is a dir. Without the trailing slash, it looks just like a normal file.

And it joined this relative link in an unexpected way.

It is recommended that the sub-dirs always have the trailing slash.

If the server recognize a directory and finds the URL missing the trailing slash, it sends a 301 redirect response to the correct URL.
